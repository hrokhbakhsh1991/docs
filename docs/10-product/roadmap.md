# Roadmap v2

## 1. Roadmap Goal

Sequence delivery of leader-centric documentation and implementation readiness without re-opening solved archive analysis.

---

## 2. Phase Plan

## Phase A: Documentation Migration Complete

Outputs:
- full English v2 doc set
- canonical framework and delta decisions
- flow and requirement consistency validation

Success condition:
- no active dependency on global marketplace discovery assumptions

## Phase B: MVP Build Readiness

Outputs:
- technical backlog from v2 docs
- API and data contract alignment
- acceptance criteria per core flow

Success condition:
- all Must-have v2 scope items mapped to implementation tasks

## Phase C: MVP Implementation

Focus:
- leader workspace operations
- dual-mode identity paths
- registration/capacity/waitlist/payment tracking core

Success condition:
- one leader can complete end-to-end operations reliably

## Phase D: Stabilization and Expansion

Focus:
- export and operational refinements
- automation candidates
- optional payment integration preparation

Success condition:
- operational friction reduced and migration-to-next-phase risks controlled

---

## 3. Deferred Track (Post-MVP)

- global discovery marketplace surfaces
- full payment gateway automation
- advanced notification orchestration
- recommendation/ranking systems
