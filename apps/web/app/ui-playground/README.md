# UI Playground (`/ui-playground`)

This route is meant for **design review** and **UI QA** only.

It renders core design-system components (buttons, inputs, cards, alerts, modal, tags) against shared tokens. Use the **Vercel Preview URL** from your pull request and append `/ui-playground` to review changes before merge.

## Draft conflict hotspots (Phase 3)

The **Conflict-ridden drafts** section lists aggregated `draft_engine.conflict` audit rows for the signed-in workspace (requires Audit read + live API). Filters: optional ISO `from` / `to`, refresh/reload.
